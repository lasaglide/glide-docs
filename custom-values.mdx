# Custom Values

Manually enter custom values. 

### Custom Values Body Text 1

When configuring your app, you often have the option to choose a **Custom Value **instead of a column value. This means you can manually enter a static value in that field. 



![Custom Values Hero](https://images.ctfassets.net/yreygbm72t08/gdrriQPKewelIjNSrqV6Y/13d2b5c59c2533ccdc9548689357b507/CleanShot_2023-12-18_at_3_.29.19_2x.png)



## Custom Values in Components

You may see the option to enter a custom value when configuring [Components](/essentials/components). When you manually enter a custom value, it will be the same every time it appears and won't change when the data in your sheet or table changes.

## Custom Values in Conditions

The value you test for in a condition defaults to an empty field where you can enter a custom value. Depending on the condition's context, you may see [Column Values](/column-values), [User Profile Columns](/user-profile-columns) or [Screen Values](/screen-values) in this menu.

If you use a custom value in the filter condition, then it will only return items that contain the custom value you are filtering for.
