# Open Link

Open a link in an in-app browser.

### Open Link Body Text 1

The Open Link action allows you to link a [component](/essentials/components) to an external webpage.



![Open Link Hero](https://images.ctfassets.net/yreygbm72t08/2ZYOEoY2KbRxulxkTDILUP/149f3196219275e79622b1be083a5f3b/image.png)



The **Open Link** action can be used as an action in the components of your app (like [Buttons](/essentials/components/button), [Collections](/essentials/components/collection), and much more), or it can be triggered along with other actions as part of a [Workflow.](/getting-started/introduction-to-workflows)

## 
**Setting up the Open Link Action**

### In the Layout Editor



### Adding Open Link Arcade

### Open Link Body Text 2

1. Click on the plus (+) sign in the **Components **panel and select the **Component** you want to trigger the action.
2. In the **General** tab on the right side of the Layout Editor, navigate to the **Actions** section and select Open Link.
3. Choose the URL you want to user to visit. You can select a column in your data that has the target link or manually enter a URL.
4. Select the icon and descriptive text to use in your app.
5. You can optionally add **Conditions** for certain criteria to be met before a user can trigger this action.

### In the Workflow Editor



### Open Link Actions Editor Arcade

### Open Link Body Text 3

1. Select the **+ New Workflow** button. If you have existing actions, click on the plus (+) sign in the left panel.
2. Select the **data source **for your action.
3. Configure your first action by clicking the first step in the workflow. You can also add conditions to your actions, if needed.
4. To add new actions to the flow, hover just underneath an action and click the plus button. In this way you can add the Open Link action as just one step in a larger workflow.
